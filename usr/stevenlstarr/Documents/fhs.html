Filesystem Hierarchy Standard

Version 1.0 • Hultrix Operating System

1. Philosophy

The Hultrix Filesystem Hierarchy Standard (HFHS) observes a Dual-Personality Architecture. The system is strictly divided into two visual and functional domains:

  • The System Layer (Lowercase): Optimized for efficiency, scripting, and administrators. Located at the root (e.g., /sys, /apps).
  • The User Layer (Capitalized): Optimized for presentation and GUI discoverability. Located within the user home (e.g., ~/Documents, ~/Applications).

2. Root Directory Structure

The root directory / is the domain of the OS administrator and is kept intentionally sparse.

Path Type Description
/apps Dir Global Application Bundles (.app). Read-only for users.
/logs Mount Persistent storage for system and service logs.
/mnt Dir Mount points for external drives, network shares, and virtual volumes.
/pub Dir Public shared files accessible to all users.
/sys Dir System Core. The immutable operating system engine.
/tmp Tmpfs Volatile RAM disk. Cleared on reboot. Holds PID/Lock files.
/usr Dir Parent directory for user home folders.

3. The System Core (/sys)

To prevent root clutter, all operating system internals are encapsulated within /sys. This directory is typically mounted as Read-Only.

/sys
├── apps      # Base system tools (flat binaries: sh, ls, cp, init)
├── boot      # Mount Point: EFI System Partition (Kernel, Initrd)
├── conf      # System-wide configuration text files
├── dev       # Mount Point: Device nodes (devtmpfs)
├── lib       # Core system libraries (libc.so, ld.so)
├── proc      # Mount Point: Process information (procfs)
└── subs      # Mount Point: Kernel subsystems (sysfs)

4. User Space

The user directory is the "product" the consumer sees. It utilizes Capitalized naming for visual hierarchy and GUI integration.

Path: /usr/username/

Directory Purpose
~/Applications User-installed App Bundles (.app).
~/Desktop Files rendered on the graphical desktop background.
~/Documents Personal files and projects.
~/Downloads Default location for web browser downloads.
~/Media Multimedia root (Music/, Video/).
~/.conf Hidden directory for per-user app configuration.

5. Application Bundles

Applications are directories ending in .app that act as files. They are self-contained and require no installation scripts.

SpaceGame.app/
├── Run               # (Executable) Entry Point script
├── meta.json         # (File) Application Manifest
├── icon.png          # (File) High-res icon
├── bin/              # Compiled binaries (hidden from user)
├── lib/              # Private libraries (.so)
└── res/              # Assets (fonts, img, shaders)

The Run Script

The entry point must setup the environment to prefer local libraries.

#!/sys/bin/sh

# Resolve absolute path
HERE="$(dirname "$(readlink -f "$0")")"

# Prioritize bundle libraries
export LD_LIBRARY_PATH="$HERE/lib:$LD_LIBRARY_PATH"

# Execute binary
exec "$HERE/bin/space_game_bin" "$@"

6. Manifest Schema (meta.json)

Every application must contain a meta.json at its root. This replaces complex Info.plist or desktop entries.

{
  "id": "com.studio.spacegame",
  "name": "Space Invaders 3D",
  "version": "1.0.5",
  "type": "Application",
  "categories": ["Game"],
  "permissions": ["network", "audio"],
  "associates": [".sav"]
}

7. SDK Integration

The Hultrix C++ SDK standardizes path resolution to abstract the filesystem layout from the developer.

namespace Hultrix::System {

    std::filesystem::path GetConfigDir() {
        // Returns: /usr/steven/.conf/com.studio.spacegame/
        return GetHomeDir() / ".conf" / GetAppID();
    }

    std::filesystem::path GetResource(std::string file) {
        // Returns: /apps/SpaceGame.app/res/{file}
        return GetBundleRoot() / "res" / file;
    }
}
